**File 1: `src/app/lib/agents/linkerAgent.ts`**

Export: `runLinkerAgent(entry: Entry, pastEntries: Entry[], synthesisOutput: SynthesisOutput): Promise<LinkerOutput>`

Implementation:
1. If pastEntries is empty, return `{ connections: [], recurringPatterns: [] }` immediately (nothing to link to).
2. Build a context string from past entries: for each past entry, include `"Entry ${entry.id} (${entry.createdAt}): ${entry.fullText.slice(0, 500)}"`. Limit to the 10 most recent past entries to stay within token limits.
3. Call OpenAI Chat Completions API via fetch:
   - Model: `gpt-4o-mini`
   - System prompt: "You are a pattern detection agent. Compare the current thought entry to past entries and find meaningful connections. Return JSON with: connections (array of objects with currentExcerpt, pastEntryId, pastExcerpt, relationship), recurringPatterns (array of patterns you notice across multiple entries). Focus on semantic connections — recurring topics, evolving ideas, contradictions, progress on goals. Be specific with excerpts."
   - User message: current entry fullText + themes from synthesisOutput + past entries context
   - `response_format: { type: "json_object" }`
   - Temperature: 0.7
4. Parse response, validate fields, return LinkerOutput.

**File 2: `src/app/lib/agents/researchAgent.ts`**

Export: `runResearchAgent(synthesisOutput: SynthesisOutput): Promise<ResearchOutput>`

Implementation:
1. Get Exa API key from `import.meta.env.VITE_EXA_API_KEY`
2. Build search queries from synthesisOutput: combine sparks and themes into 2-3 search queries (max). Use the most interesting/specific sparks first, fall back to themes.
3. For each query, call Exa Search API via fetch:
   - URL: `https://api.exa.ai/search`
   - Method: POST
   - Headers: `Authorization: Bearer ${apiKey}`, `Content-Type: application/json`
   - Body: `{ query: string, numResults: 3, type: "neural", contents: { text: { maxCharacters: 300 } } }`
4. Combine results from all queries into a flat array of references.
5. Map each Exa result to: `{ title: result.title, url: result.url, snippet: result.text, relevantTo: querySource }` where querySource is the spark/theme that generated the query.
6. Deduplicate by URL.
7. Return ResearchOutput with up to 8 references total.

Error handling for Exa: If the API key is missing or Exa returns an error, return `{ references: [] }` with a console.warn (don't crash the pipeline — research is nice-to-have).
